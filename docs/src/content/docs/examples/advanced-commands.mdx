このページでは、esa-cliの高度な使用例と実践的なシナリオを紹介します。基本的な使い方は[基本的なコマンド例](/examples/basic-commands/)を、詳細な説明は[ユーザーガイド](/guides/user-guide/)を参照してください。

## 🔄 一括処理

### 記事の一括バックアップ

```bash
# 特定のカテゴリの記事を一括バックアップ
esa-cli list --category "開発" | xargs -I {} esa-cli download {} --output ./backup

# 特定のタグが付いた記事を一括バックアップ
esa-cli list --tag "重要" | xargs -I {} esa-cli download {} --output ./backup
```

### カテゴリの一括移動

```bash
# カテゴリ内の記事を別のカテゴリに移動
esa-cli list --category "古いカテゴリ" | xargs -I {} esa-cli update {} --category "新しいカテゴリ"
```

## 📦 バックアップとリストア

### 定期的なバックアップ

```bash
# 毎日のバックアップを実行するスクリプト
#!/bin/bash
DATE=$(date +%Y%m%d)
BACKUP_DIR="./backup/$DATE"
mkdir -p "$BACKUP_DIR"
esa-cli list | xargs -I {} esa-cli download {} --output "$BACKUP_DIR"
```

### 特定の期間の記事をバックアップ

```bash
# 過去30日間の記事をバックアップ
esa-cli list --days 30 | xargs -I {} esa-cli download {} --output ./backup
```

## 🔍 高度な検索とフィルタリング

### 複合条件での検索

```bash
# カテゴリとタグを組み合わせた検索
esa-cli search "キーワード" --category "開発" --tag "API"

# 日付範囲を指定した検索
esa-cli search "キーワード" --from "2024-01-01" --to "2024-03-31"
```

### 検索結果の加工

```bash
# 検索結果をMarkdown形式で保存
esa-cli search "キーワード" --format markdown > search_results.md

# 検索結果から記事を一括ダウンロード
esa-cli search "キーワード" | xargs -I {} esa-cli download {}
```

## 🔧 カスタマイズと自動化

### カスタムエディタの使用

```bash
# 特定のエディタで記事を編集
esa-cli edit 123 --editor "code"

# 環境変数でデフォルトエディタを設定
export ESA_CLI_EDITOR="code"
```

### シェルスクリプトとの連携

```bash
# 記事の更新を監視して通知
#!/bin/bash
while true; do
    esa-cli list --category "開発" | while read -r article; do
        if [ "$(esa-cli show "$article" --format json | jq '.updated_at')" > "$LAST_CHECK" ]; then
            notify-send "記事が更新されました: $article"
        fi
    done
    sleep 3600
done
```

## 📊 レポート生成

### 記事の統計情報

```bash
# カテゴリごとの記事数を集計
esa-cli list --format json | jq 'group_by(.category) | map({category: .[0].category, count: length})'

# タグごとの記事数を集計
esa-cli list --format json | jq 'map(.tags) | flatten | group_by(.) | map({tag: .[0], count: length})'
```

### アクティビティレポート

```bash
# 過去30日間の更新履歴を取得
esa-cli list --days 30 --format json | jq 'map({title: .title, updated_at: .updated_at})'
```

これらの例は、必要に応じてカスタマイズして使用できます。より詳細な説明やオプションについては、[ユーザーガイド](/guides/user-guide/)の該当するセクションを参照してください。
